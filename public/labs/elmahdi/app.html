Hint: JSON Injection via Querystring

The page fetches a config endpoint that reflects the v query parameter into the response JSON without escaping. You can break out of the JSON structure and inject new top-level keys. One of those keys gets merged into window.
1
Probe the config endpoint ›
Try /elmahdi/api/config?v=test directly in your browser. See how v is reflected into the JSON response. Now try v=test" — does the quote come back escaped?
2
Understand the parse path ›
The client does JSON.parse(rawText). Your payload must keep the JSON valid while injecting new structure. That means balancing braces and brackets.
3
Find the dangerous sink ›
Search the bootstrap script for something like Object.assign(window, config.window). If you can inject a "window" key into the config, its contents get merged into window.
4
What to overwrite on window ›
Assigning to window.location (as a string) navigates the browser to that URL. javascript: URIs execute code: window.location = "javascript:alert(1)".
5
Plan the structural breakout ›
The template is: 
{
  "version": "YOUR_INPUT",
  "features": { ... },
  "api": { ... }
}
You need to close the version string, close the outer object, inject a comma, add "window": { "location": "javascript:..." }, then re-open something so the original trailing JSON is still syntactically valid when concatenated. A common trick: wrap the rest in a dummy key whose value starts opening a string that never closes, then let the trailing " from the template close it.

Navigate to:

/elmahdi/app?v=1","window":{"location":"javascript:alert('JSON_INJECTION')"},"x":"

The resulting JSON body becomes:

{
  "version": "1","window":{"location":"javascript:alert('JSON_INJECTION')"},"x":"",
  "features": {...},
  "api": {...}
}

This is valid JSON. config.window is now {"location":"javascript:..."}, and Object.assign(window, config.window) sets window.location, triggering navigation to the javascript: URI.
