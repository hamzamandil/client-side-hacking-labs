Hint: BuzzBoard (insertAdjacentHTML)

The app has a sanitize() function. It's used in most render paths — but not all. Find a path where hash params reach a DOM API without sanitization.
1
Find the source›
Open social-feed.js. Look at handleDeepLinks(). It parses URL query AND hash fragments. Key hash params: bio, announcement.
2
Trace each hash param to its sink›
announcement goes to showAnnouncement which uses textContent — SAFE. bio goes to updateBio which uses insertAdjacentHTML and innerHTML directly — NOT SAFE.
3
The two branches in updateBio› 
updateBio: function(bio) {
  var bioEl = this.profileContainer.querySelector('.bb-profile-bio');
  if (bioEl) {
    bioEl.insertAdjacentHTML('beforeend', '...'+bio+'...');
  } else {
    tempDiv.innerHTML = '...'+bio+'...';
    this.profileContainer.appendChild(tempDiv);
  }
}
Both paths are sinks. The "else" path fires when the profile hasn't loaded yet.
4
Trigger the right path›
If you provide ?profile=victim, a profile is rendered and .bb-profile-bio exists — the insertAdjacentHTML path fires. If you skip profile, the else branch fires. Both exploit the same bio payload.

URL payload:

/labs/xss-hunt/hunt3-insertadjacenthtml.html?profile=victim#bio=<img src=x onerror=alert(1)>

Flow: profile loads → .bb-profile-bio exists → updateBioFromHash reads #bio= → insertAdjacentHTML('beforeend', ...bio...) with raw HTML → img onerror fires.
