Hint: TicketFlow jQuery Sink

The search code has TWO rendering paths. One uses an escape helper, the other doesn't. Your job: pick the unsafe path and control what flows into it.
1
Find the source ›
Open /js/apps/ticket-app.js in DevTools (Sources tab) or view it directly. Search for where the query comes from — TicketFlow.getQueryParam. The query is read from the URL ?q= parameter on page load and also reflects on hash fragments. The init function auto-runs a search if ?q= is present.
2
Trace the query into render ›
The search calls fetch('/api/search?q=...'), then hands the response plus the original query to renderResults(data, query). Read renderResults carefully — the query appears in TWO places.
3
Compare the two paths ›
When results exist, the code uses TicketFlow.escapeText(query) — safe. When results are empty, the code uses $(container).html(...query...) directly — no escaping. That's your sink. 
if (!data.results || data.results.length === 0) {
  $(container).html(
    '...No results matching "<strong>' + query + '</strong>"...'
  );
  return;
}
4
How to reach the empty path ›
The backend returns results when the query matches. Use a query that matches nothing real — e.g. zzzpayload. Then append your HTML payload. The whole string goes through .html().
5
jQuery .html() quirks ›
jQuery's .html() strips <script> tags but executes event handlers on other elements. Use <img src=x onerror=...> or <svg onload=...>. The img fails to load → onerror fires.
6
Deliver the payload ›
Put the payload directly in the URL: ?q=ZZZ<img src=x onerror=alert(1)>. The init function reads the URL, calls execute, hits the backend (no results because "ZZZ..." doesn't match), falls into the empty path, and .html() renders your img tag.

Navigate to:

/labs/xss-hunt/hunt1-jquery-sink.html?q=zzznoresults<img src=x onerror=alert('XSS')>

Flow:

1. Search.init() reads ?q= from URL

2. Sets the search input value and calls execute(query)

3. Fetches /api/search — no tickets match "zzznoresults…" so results is empty

4. Empty path in renderResults runs $(container).html('...' + query + '...')

5. jQuery parses the HTML, creates the <img>, browser tries to load src=x, fails, fires onerror → alert

Why the safe path is safe: the code uses TicketFlow.escapeText(query) there. The developer remembered the "success" path but forgot the "empty" path.
