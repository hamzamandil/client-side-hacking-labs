DOM XSS #4: Template Literal Breakout Medium

Objective

Execute alert('XSS') by breaking out of a JavaScript template literal (backtick string).

The app uses ES6 template literals with ${} interpolation. Your input goes inside a template literal that builds HTML.

Hints

Template literals use backticks (`) and ${expression} for interpolation. The expression inside ${} is evaluated as JavaScript. If you can inject ${}, you can run code.
Your input is placed directly into a template literal like: `<p>Hello ${userInput}</p>`. If the input contains ${...}, the content is evaluated as JS during string construction.
Try the input: ${alert('XSS')} — This gets evaluated as a JavaScript expression inside the template literal. Also try: ${constructor.constructor('alert(1)')()} for a more creative approach.

Vulnerable Code

var userInput = getParam('greeting');

// VULNERABLE: User input inside eval'd template
var html = eval(
  '`<div class="card">
    <h3>Greeting Card</h3>
    <p>${userInput}</p>
  </div>`'
);
container.innerHTML = html;

Real-world context: Some apps use eval() with template literals to build dynamic HTML. This is common in template engines, admin dashboards, and code that migrated from older template systems. The ${} syntax is evaluated as JavaScript, making it a code execution vector.

Bug bounty tip: Search for eval(` or new Function(` with user-controlled data flowing into the template. This is less common than innerHTML but much more powerful since it's direct code execution.
