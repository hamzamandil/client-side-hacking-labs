DOM XSS #2: document.write Injection Easy

Objective

Execute alert('XSS') by exploiting the document.write() sink that uses a URL query parameter.

Hints

The page reads the ?name= query parameter from the URL. Try adding ?name=test to see it reflected.
The parameter is placed inside an HTML attribute in a document.write() call: <img src="..." alt="VALUE">. You need to break out of the attribute.
Try: ?name=" onload="alert('XSS')" x=" or ?name="><img src=x onerror=alert('XSS')> — break out of the alt attribute and inject your own event handler or tag.

Vulnerable Code

// Source: URL query parameter
var params = new URLSearchParams(
  window.location.search
);
var name = params.get('name') || 'Guest';

// Sink: document.write (dangerous!)
document.write(
  '<div class="welcome-card">' +
  '<img src="/img/avatar.png" alt="' + name + '">' +
  '<h2>Welcome, ' + name + '!</h2>' +
  '</div>'
);

👤 User Welcome Page

Real-world context: document.write() is a legacy API still found in many older apps and third-party widgets. It writes directly to the document during parsing, making it a powerful XSS sink. The key insight is that you need to break out of the HTML context (attribute, tag) the input is placed into.
