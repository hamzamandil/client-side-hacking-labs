DOM XSS #3: jQuery Sink Exploitation Medium

Objective

Execute alert('XSS') by bypassing the input filter and exploiting the jQuery .html() sink.

The app has a "sanitizer" that tries to block XSS. Find a way around it.

Hints

The filter uses a regex to remove <script> tags and on*= event handlers. But regex-based HTML sanitization is almost always bypassable.
The regex uses case-insensitive matching for script tags, but what about the onerror handler? Check if the regex handles mixed case or newlines in event handlers.
Try: <img src=x oNeRrOr=alert('XSS')> (mixed case) or <svg/onload=alert('XSS')> (different tag) or <details open ontoggle=alert('XSS')>. The filter only strips specific patterns, not all vectors.

Vulnerable Code

function sanitize(input) {
  // Remove script tags
  input = input.replace(
    /<script\b[^<]*(?:(?!<\/script>)<[^<]*)*<\/script>/gi,
    ''
  );
  // Remove onerror/onload handlers
  input = input.replace(
    /\bon(error|load)\s*=/gi, ''
  );
  return input;
}

// Sink: jQuery .html()
$('#comment-display').html(
  sanitize(userInput)
);

Real-world context: Many apps try to allow "safe HTML" (bold, italic, links) while blocking dangerous tags. Regex-based filters are notoriously brittle. jQuery's .html() is equivalent to innerHTML — it parses and executes any HTML it receives. In the real world, look for any jQuery .html(), .append(), .prepend(), .after(), .before() calls with user-controlled data.
