Marketplace Extension

Click "Load Widget" to initialize the marketplace extension iframe.

The extension receives an access token via postMessage to perform API calls on your behalf.

// Token exchange log

Hint: Token Forwarding via Wildcard postMessage

The app forwards auth tokens to extension iframes using postMessage(tokenData, "*"). The wildcard origin means ANY window can request and receive the token. Based on Microsoft's MSRC research.
1
Beautify and read the code›
The minified block initializes window._azHost. It has getToken(), sendTokenToExtension(), and listenForRequests(). The message handler listens for az:auth:request and az:extension:ready.
2
Find the targetOrigin›
In sendTokenToExtension, look at the postMessage call: 
targetWindow.postMessage({
  type: "az:auth:token",
  accessToken: tokenData.accessToken,
  ...
}, "*");  // WILDCARD!
Token is sent to ANY origin.
3
The "source check" is broken›
The handler checks e.source === self.iframeRef. But in the else branch, it sends the token anyway! The dev logged "Source check: comparing..." but still calls sendTokenToExtension(e.source) regardless.
4
How to intercept the token›
Two approaches:
(a) Open this page via window.open(), then send {type:'az:extension:ready'} — the host fetches a token and sends it back to your window
(b) Or just postMessage({type:'az:auth:request'}) from this page's console — the token is logged and visible in the token log panel

Quick test (in this page's console):

// Listen for the token response
window.addEventListener('message', e => {
  if (e.data && e.data.type === 'az:auth:token') {
    alert('STOLEN TOKEN: ' + e.data.accessToken.substring(0, 40));
  }
});
// Request the token
postMessage({type:'az:auth:request'}, '*');
