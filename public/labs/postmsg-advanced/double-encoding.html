Note: The server strips < > " ' = from the ref parameter before reflecting it. This prevents standard XSS payloads. Or does it?

Hint: Double URL-Encoding Context Mismatch

The server reflects ref inside a JS string in a <script> tag. It strips < > " ' = to "prevent XSS". But the filter runs on the URL-decoded value ONCE, while the browser may decode TWICE. Based on the Intigriti CTF and jub0bs chain.
1
Understand the reflection context›
The server responds with HTML where ref is inside: 
<script>
  console.log("Event tracked: pageview, ref: YOUR_INPUT");
</script>
You need to break out of the JS string. But " is stripped by the server.
2
What the filter does›
Server code: safeRef = ref.replace(/[<>"'=]/g, ''). It removes these characters from the decoded value. Standard payloads like "; alert(1); // lose the " and become ; alert(1); // which doesn't break the string.
3
Double encoding trick›
What if the quote is double-encoded? %2522 = %22 after first decode = " after second decode. The server's filter sees %22 (not a literal quote) and lets it through. But when the browser renders the HTML response, it decodes it again into ".

Wait — does Express double-decode? Not by default. But the encodeURIComponent on the client side and the server's query parser interact in a specific way. Test what happens with %2522 vs the direct client encodeURIComponent flow.
4
Alternative: JS escape sequences›
You're inside a JS string literal. JS allows \x22 (hex escape) and \u0022 (unicode escape) for quotes. The server strips literal " but NOT \x22 or \u0022 — those are just alphanumeric + backslash. Try: \x22; alert(1); //
5
But backslash in URLs›
\ needs to survive the URL encoding roundtrip. In the input field, type the literal text \x22; alert(1); // — the client does encodeURIComponent which preserves it as %5Cx22..., the server decodes to \x22, filter doesn't strip it (no literal "), and it reaches the JS context where the JS engine parses \x22 as a quote.

In the ref input field, type:

\x22; alert(\x27XSS\x27); //

Click "Load Tracker". The server strips nothing (no literal " or '). The rendered script becomes:

console.log("Event tracked: pageview, ref: \x22; alert(\x27XSS\x27); //");

The JS engine parses \x22 as ", closing the string. alert(\x27XSS\x27) fires. The // comments out the rest.
