Chain #4: The Grand Chain (CSPT+PostMessage+OAuth=ATO) Expert

Objective

The ultimate challenge. Chain FOUR vulnerability classes into a single account takeover attack. This is what separates elite bug bounty hunters from script kiddies.

Chain: CSPT → Response Injection → PostMessage → OAuth Token Theft → ATO

Hints

CSPT: The dashboard fetches config from /api/settings/{section} — section param is user-controlled
Response Injection: Upload malicious JSON to /api/upload, traverse to it
PostMessage: The config response can contain a widgetUrl that gets loaded in an iframe, which communicates via postMessage
OAuth: The postMessage handler initiates SSO flows based on iframe messages
1. Upload malicious config JSON with a widgetUrl pointing to your attacker page
2. CSPT: Make dashboard load your config instead of the real one
3. Your attacker page loads in the iframe and sends a postMessage
4. The postMessage triggers an OAuth flow with manipulated redirect_uri
5. OAuth authenticates the victim and redirects with token
6. Use the session to change email/password → ATO
For this lab, you can simplify by going step by step:
1. Upload config: {"section":"x","data":{},"widgetUrl":"/labs/attacker/chain-widget.html","autoApply":[{"endpoint":"/api/user/change-email","method":"POST","body":{"email":"attacker@evil.com"}}]}
2. Load with CSPT: ?section=../files/{fileId}
3. The autoApply triggers the email change
4. The widget iframe sends postMessage to confirm
OR just chain the CSPT2CSRF from lab 4 with the OAuth from lab 3!
Honestly, for this lab, the easiest path is:
1. Login as victim
2. Upload JSON: {"section":"general","data":{},"autoApply":[{"endpoint":"/api/user/change-email","method":"POST","body":{"email":"attacker@evil.com"}},{"endpoint":"/api/user/change-password","method":"POST","body":{"newPassword":"hacked"}}]}
3. CSPT to load it: ?section=../files/{id}
This gives you CSPT → Response Injection → CSRF → ATO in one chain!
