Chain #4: The Grand Chain — CSPT + PostMessage + OAuth = ATO Expert

Objective

Chain three individually low-severity bugs into a full account takeover. Steal the victim's OAuth access token without any direct interaction beyond visiting one link.

The Three Bugs

Bug A · CSPT Marketplace page fetches /api/integrations/{id}/manifest. The id comes from ?integration= with no validation.
Bug B · OAuth Server supports response_type=token (implicit flow). Returns access token in URL fragment.
Bug C · postMessage Gadget OAuth callback error page has an analytics handler: responds to getAnalytics with full page URL (including fragment).

How the Chain Fits

Upload a manifest via /api/upload. Then use the CSPT: ?integration=../../files/{fileId}. Browser normalizes to /api/files/{fileId}. App blindly trusts the response and iframes oauthUrl.
In the manifest, set oauthUrl to /oauth/authorize?client_id=vuln-app&response_type=token&redirect_uri=/oauth/callback&state=x. Victim's session auto-approves — token lands in fragment.
The callback handler shows an error page if the error query param is set. But you can also just hit /oauth/callback directly — the fragment carries through. The error page has the postMessage gadget.
Parent page sends {type:'getAnalytics'} to the iframe. Gadget responds with {type:'analyticsData', url: location.href}. Parse #access_token= from the URL. Done.

// Marketplace page

  .then(m => {

  });
