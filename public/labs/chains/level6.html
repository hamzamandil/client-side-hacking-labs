Chain #6: CORS + XSS = Session Hijack Hard

Objective

The main API trusts *.localhost origins (CORS). A "blog" subdomain page has DOM XSS. Chain: exploit the XSS on the blog to make a credentialed CORS request to the API and steal the victim's secrets.

Hints

Bug 1: The blog page at the bottom has innerHTML XSS via the ?post= parameter.
Bug 2: /api/cors-demo/subdomain-data trusts any *.localhost origin with credentials.
Neither bug alone is critical. Together: CRITICAL.
Craft a ?post= value that injects a script to fetch the CORS-protected API with credentials and exfiltrate the data. The XSS runs on the blog's origin, which is trusted by CORS.
?post=<img src=x onerror="fetch('/api/cors-demo/secrets',{credentials:'include'}).then(r=>r.json()).then(d=>document.getElementById('blog-post').innerHTML=JSON.stringify(d))">
The XSS on the "subdomain" makes a credentialed fetch. Since the blog origin is trusted by CORS, the browser allows reading the response.
