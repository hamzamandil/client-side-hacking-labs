Chain #3: PostMessage + OAuth = ATO Expert

Objective

Full ATO chain: Exploit a postMessage listener on the app to initiate a rogue OAuth flow, steal the resulting token, then use it to change the victim's email and password. This combines postMessage exploitation with OAuth token theft.

Hints

The app's main page has a postMessage handler for "SSO integration" that accepts {type:'sso_redirect', url:'...'} messages and navigates the user to the URL. No origin check. This is your entry point.
Use the postMessage handler to redirect the victim to an OAuth URL with a manipulated redirect_uri (using path traversal). The OAuth flow will authenticate the victim and redirect them with the token to your attacker page.
1. Open victim page in iframe/popup
2. Send postMessage: {type:'sso_redirect', url:'/oauth/authorize?client_id=vuln-app&response_type=token&redirect_uri=...(traversal to steal page)'}
3. Victim gets redirected through OAuth → your steal page
4. Steal page captures the token
5. Use the session cookie set during OAuth to change email/password
