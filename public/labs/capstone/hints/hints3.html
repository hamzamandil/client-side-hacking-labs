DevPortal - Hints

Only reveal what you need. Try to solve it yourself first.

Open DevTools Network tab and reload the page. Watch what API requests are made. One of them loads settings from a path that includes a URL parameter. What parameter controls the path?
The page fetches /api/settings/{section} where section comes from the ?section= URL parameter (defaults to "general"). The value is concatenated directly into the fetch URL without sanitization. This is Client-Side Path Traversal.
After fetching settings, the code checks for a title field and renders it via innerHTML. It also checks for an autoApply array and executes each action as an authenticated fetch request. If you control the response, you control what actions are performed.
Use the /api/upload endpoint to upload malicious JSON. Send a POST to /api/upload with {"content": "your JSON here", "contentType": "application/json"}. It returns a file URL like /api/files/abc123. Then use CSPT to make the settings fetch load your file instead.
1. Upload JSON: POST /api/upload with content containing a title field with XSS payload, or an autoApply array with email change action.
2. Note the returned fileId.
3. Visit the DevPortal with ?section=../files/{fileId}.
The fetch resolves to /api/settings/../files/{fileId} = /api/files/{fileId} — your poisoned response.
