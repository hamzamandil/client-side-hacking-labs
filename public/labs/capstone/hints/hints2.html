ChatConnect - Hints

Only reveal what you need. Try to solve it yourself first.

The chat widget isn't just a visual component. Open DevTools and look at the JavaScript. The widget listens for something beyond user chat input. Search the source for addEventListener.
The widget has a window.addEventListener('message', ...) handler — it accepts postMessage events. Read the handler carefully. What actions does it support? What origin check does it use?
The origin check uses indexOf('localhost') which is weak — but since you're already on localhost, your messages pass anyway. The handler supports three actions: updateWidget (innerHTML sink), setAgentName (innerHTML sink), and execCommand (eval sink).
You need to send a postMessage to the page. You can do this from the browser console, from the attacker page at /labs/attacker/postmessage-attacker.html, or by opening the capstone in a new window and sending messages from another tab. The message format is JSON with an action field.
Open the browser console on the ChatConnect page and try: window.postMessage(JSON.stringify({action:'execCommand',cmd:'alert("XSS")'}), '*'). The execCommand action passes your cmd string directly to eval().
