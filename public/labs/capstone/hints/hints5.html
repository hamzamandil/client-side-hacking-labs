SocialHub - Hints

This is an ATO challenge. You need to chain bugs to change the victim's email. Only reveal what you need.

There are TWO bugs to find. One is a DOM XSS, the other is a missing security control. Search the page, interact with the features (search, profile, settings), and read the JavaScript source. Both bugs are in the client-side code.
The search feature takes user input and renders it via innerHTML without sanitization. Try the search box or the ?q= URL parameter. Your HTML will render directly in the search results area.
The profile settings page makes two API calls when you save: POST /api/user/profile/update and POST /api/user/change-email. Neither uses CSRF tokens. If you can execute JavaScript in the victim's browser context, you can call these endpoints with credentials: 'include'.
Chain: DOM XSS + CSRF = ATO. Craft a URL with a ?q= parameter containing an XSS payload. The XSS payload should make a fetch() call to /api/user/change-email with {email: 'attacker@evil.com'} and credentials: 'include'. When the logged-in victim clicks this link, their email changes.
1. Log in as victim
2. Visit: ?q=<img src=x onerror="fetch('/api/user/change-email',{method:'POST',headers:{'Content-Type':'application/json'},credentials:'include',body:JSON.stringify({email:'attacker@evil.com'})})">
3. The XSS fires, the email change request goes out with the victim's cookies, and the email changes to attacker@evil.com. Lab solves when the email is changed.
