CSPT #4: CSPT to CSRF (CSPT2CSRF) Hard

Objective

Change the victim's email to attacker@evil.com by chaining CSPT with CSRF. The settings page auto-loads config from an API and auto-applies settings. Poison the config response to trigger an email change.

You must: 1) Log in as "victim" 2) Upload malicious config 3) CSPT to make the settings page load your config 4) The auto-apply triggers email change

Hints

The settings page fetches /api/settings/{section} and if the response contains an autoApply field with actions, it executes them. Actions can include API calls to change email, password, etc. The page uses cookies for auth (credentials: 'include').
Upload JSON like: {"section":"general","data":{"theme":"dark"},"autoApply":[{"endpoint":"/api/user/change-email","method":"POST","body":{"email":"attacker@evil.com"}}]}. This makes the auto-apply mechanism call the email change endpoint.
1. Log in as victim (victim/password123)
2. Upload the malicious config JSON
3. Navigate to this page with ?section=../files/{fileId}
4. The page fetches your poisoned config
5. Auto-apply executes the email change
6. Check the CSRF log to confirm

Attack Flow

// 1. Page loads settings config
fetch('/api/settings/' + section)
  .then(r => r.json())
  .then(config => {
    renderSettings(config);

    // 2. If config has autoApply, execute actions
    if (config.autoApply) {
      config.autoApply.forEach(action => {
        // 3. Makes authenticated requests!
        fetch(action.endpoint, {
          method: action.method,
          headers: {'Content-Type': 'application/json'},
          credentials: 'include',
          body: JSON.stringify(action.body)
        });
      });
    }
  });

Real-world context: CSPT2CSRF is one of the most impactful client-side attacks. When an app:

  1. Fetches configuration/data from an API endpoint using a user-controlled path
  2. Uses the response to drive state-changing actions (auto-save, auto-apply, form submission)
  3. Includes cookies/auth in those state-changing requests

Then an attacker can poison the response via CSPT and make the victim perform any action the API supports — email change, password change, account deletion, fund transfer.
