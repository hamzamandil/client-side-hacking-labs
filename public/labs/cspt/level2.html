CSPT #2: Filter Bypass with Backslash Medium

Objective

The app strips forward slashes from the path parameter to prevent CSPT. Bypass the filter using backslashes or double-encoding to reach /api/user/profile/admin.

Hints

The code runs userId.replace(/\//g, '') which removes all forward slashes. So ../user/profile becomes ..userprofile. But what about backslashes?
Modern browsers treat backslashes (\) as equivalent to forward slashes in URL paths. So ..\ works the same as ../. The filter only removes / but not \.
Try: ?userId=..\user\profile\admin. The filter won't remove the backslashes, and the browser resolves /api/settings/..\user\profile\admin to /api/user/profile/admin. Note: you may need to URL-encode the backslashes as %5c.

Vulnerable Code

var userId = params.get('userId');

// "Security" filter: remove forward slashes
userId = userId.replace(/\//g, '');

// But backslashes still work as path separators!
fetch('/api/settings/' + userId)
  .then(r => r.json())
  .then(renderData);

Real-world context: The Opera browser cashback feature had this exact bug — an $8,000 bounty. The developer filtered forward slashes but forgot backslashes. Browsers normalize \ to / in URL paths before sending HTTP requests. Other bypass techniques:

  • \ instead of / (backslash)
  • %2f URL-encoded slash (if filter runs before decode)
  • %252f double-encoded slash
  • %5c URL-encoded backslash
  • ..;/ semicolon trick (some servers)
