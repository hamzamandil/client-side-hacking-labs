Advanced #1: Prototype Pollution to XSS Hard

Objective

Execute alert('XSS') by exploiting a prototype pollution vulnerability in the URL parameter parser. Pollute Object.prototype to override a config value that flows into a script src attribute.

Hints

JavaScript objects inherit from Object.prototype. If you can set properties on it (e.g., Object.prototype.polluted = true), then every object will have .polluted === true unless it defines its own. A vulnerable parser lets you set __proto__ properties via URL params.
The page uses a custom deparam() function that converts URL query strings into objects. It doesn't block __proto__ keys. Try: ?__proto__[polluted]=yes and check ({}).polluted in the console.
After parsing params, the code reads config.scriptSrc || defaults.scriptSrc. If neither config nor defaults has scriptSrc, it falls back to Object.prototype.scriptSrc (your polluted value!). Try: ?__proto__[scriptSrc]=data:text/javascript,alert('XSS')

Vulnerable Code

// Vulnerable URL param parser
function deparam(qs) {
  var obj = {};
  qs.split('&').forEach(pair => {
    var [path, val] = pair.split('=');
    var keys = path.split(/[\[\]]+/).filter(Boolean);
    var cur = obj;
    keys.forEach((k, i) => {
      if (i === keys.length - 1) cur[k] = val;
      else cur = cur[k] = cur[k] || {};
    });
  });
  return obj;
}

// Gadget: loads script from config
var src = config.scriptSrc;
if (src) {
  var s = document.createElement('script');
  s.src = src;
  document.body.appendChild(s);
}

Real-world context: This is based on the HubSpot tracking script prototype pollution finding. The jQuery deparam() function (and many similar URL parsers) didn't block __proto__ keys. Combined with "gadgets" — code that reads from object properties and uses them in sinks like script.src, innerHTML, or eval — prototype pollution becomes XSS.
