Advanced #3: setTimeout String Execution Easy

Objective

Execute alert('XSS') by injecting code through a setTimeout() call that uses a string argument instead of a function. The delay value comes from a URL parameter.

Hints

setTimeout("code", ms) works like eval("code") — it executes the string as JavaScript after the delay. This is a legacy pattern still found in older codebases.
The page reads ?action= from the URL and passes it to setTimeout("performAction('" + action + "')", 1000). You need to break out of the string context in the setTimeout call.
Try: ?action=');alert('XSS');// — This closes the function call, adds your alert, and comments out the rest. The setTimeout string becomes: performAction('');alert('XSS');//')

Vulnerable Code

var action = params.get('action') || 'welcome';

// VULNERABLE: string-based setTimeout
setTimeout(
  "performAction('" + action + "')",
  1000
);

Real-world context: setTimeout(string) and setInterval(string) are equivalent to eval(). Modern linters flag this, but legacy code — especially jQuery plugins, ad scripts, and tracking code — still uses string-based timers. Look for this pattern in minified third-party scripts where developers passed user input into timer callbacks.
