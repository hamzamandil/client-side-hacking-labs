Advanced #2: createContextualFragment XSS Medium

Objective

Execute alert('XSS') by bypassing the sanitizer. The page strips <script> tags and common event handlers, then uses createContextualFragment() to insert HTML. This sink re-parses and executes handlers the regex missed.

Hints

document.createRange().createContextualFragment(html) creates a DocumentFragment by parsing the HTML string. Unlike textContent, it DOES execute event handlers on the parsed elements when they're inserted into the DOM.
The sanitizer only removes onerror, onload, and onclick. There are 60+ event handlers in HTML: onfocus, onmouseover, ontoggle, onauxclick, etc. Try handlers the regex doesn't cover.
Try: <details open ontoggle=alert('XSS')>test</details> or <input onfocus=alert('XSS') autofocus> or <body onpageshow=alert('XSS')>. The sanitizer won't catch these event handlers.

Vulnerable Code

function sanitize(html) {
  html = html.replace(/<script[\s\S]*?<\/script>/gi, '');
  html = html.replace(/\bon(error|load|click)\s*=/gi, '');
  return html;
}

var range = document.createRange();
var frag = range.createContextualFragment(
  sanitize(userInput)
);
container.appendChild(frag);

Real-world context: This is based on the HubSpot inlineMessage DOM XSS finding (HIGH severity, 50+ bug bounty targets). The HubSpot form widget used createContextualFragment() to render the inline success message. Even though some handlers were stripped, the range of possible HTML event handlers is enormous. This sink is often overlooked because developers assume only innerHTML and eval() are dangerous.
